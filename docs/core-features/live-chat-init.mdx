# Live Chat Initialization

Start a live chat session with your users.

## Updated Usage

Now, you must pass a config object:

```js
sdk.initChat({
  token: 'YOUR_API_TOKEN',
  userId: 'USER_ID', // Recommended for user tracking
  userEmail: 'user@example.com', // Optional user email
});
```

- `token`: Required API token
- `userId`: Recommended user identifier. Essential for robust user tracking. The SDK no longer defaults to null if omitted.
- `userEmail`: Optional user email for enhanced identification. If provided, overrides user email from other sources.

### Example Use Case
- Start a chat for a logged-in user with their ID

### Edge Cases
- If `userId` is omitted, chat will still start but user tracking will be disabled. Providing a `userId` is highly recommended for full feature functionality.
- Passing a string instead of an object will throw an error
